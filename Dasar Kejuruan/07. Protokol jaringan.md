## 1. TCP/IP (Transmission Control Protocol / Internet Protocol)  

### ðŸ”¹ Pengertian
TCP/IP adalah **sekumpulan protokol standar** yang digunakan untuk menghubungkan komputer dalam jaringan internet maupun intranet.  
Protokol ini menjadi **dasar komunikasi data di jaringan komputer**.  

### ðŸ”¹ Fungsi
- Mengatur cara data dikirim, diterima, dan diatur dalam jaringan.  
- Menjamin data sampai ke tujuan dengan benar (reliability).  
- Menyediakan alamat logis (IP Address) untuk identifikasi perangkat.  

### ðŸ”¹ Lapisan TCP/IP

# ðŸ“˜ Lapisan-Lapisan Protokol TCP/IP

Model **TCP/IP** memiliki 4 lapisan utama (sering dibandingkan dengan 7 lapisan OSI).  
Setiap lapisan memiliki fungsi dan contoh protokol yang berbeda.

---

## 1. Application Layer
ðŸ‘‰ Lapisan ini paling dekat dengan **pengguna**.

### ðŸ”¹ Fungsi
- Menyediakan layanan aplikasi agar pengguna bisa berkomunikasi melalui jaringan.
- Mengatur bagaimana aplikasi mengakses jaringan.

### ðŸ”¹ Protokol
- **HTTP/HTTPS** â†’ membuka website
- **DNS** â†’ mencari alamat IP dari nama domain
- **FTP** â†’ transfer file
- **SMTP/POP3/IMAP** â†’ email

### ðŸ”¹ Contoh
Membuka **YouTube** di browser â†’ request video dikirim lewat HTTP.

---

## 2. Transport Layer
ðŸ‘‰ Mengatur **pengiriman data antar aplikasi**.

### ðŸ”¹ Fungsi
- Memecah data menjadi paket kecil
- Menjamin keandalan data (error checking, retransmisi)
- Mengatur port komunikasi (aplikasi mana yang menerima data)

### ðŸ”¹ Protokol
- **TCP (Transmission Control Protocol)** â†’ handal, untuk browsing, email
- **UDP (User Datagram Protocol)** â†’ cepat, untuk game online, streaming

### ðŸ”¹ Contoh
Main game online â†’ pakai **UDP** agar lebih cepat walau ada kemungkinan paket hilang.

---

## 3. Internet Layer
ðŸ‘‰ Mengatur **alamat dan jalur data**.

### ðŸ”¹ Fungsi
- Menentukan alamat tujuan (IP Address)
- Menentukan rute paket data melalui jaringan (routing)

### ðŸ”¹ Protokol
- **IP (Internet Protocol)** â†’ alamat IP perangkat
- **ICMP (Internet Control Message Protocol)** â†’ cek koneksi (`ping`)
- **ARP (Address Resolution Protocol)** â†’ mencari alamat MAC dari alamat IP

### ðŸ”¹ Contoh
Ketik `google.com` â†’ komputer mencari IP server Google â†’ paket dikirim ke alamat tersebut.

---

## 4. Network Access Layer (Link Layer)
ðŸ‘‰ Lapisan paling bawah, berhubungan langsung dengan **hardware jaringan**.

### ðŸ”¹ Fungsi
- Mengatur transmisi data melalui media (kabel, Wi-Fi, fiber)
- Mengatur format frame agar dipahami perangkat fisik

### ðŸ”¹ Protokol/Teknologi
- **Ethernet (LAN)**
- **Wi-Fi (WLAN)**
- **PPP, DSL, Fiber Optic**

### ðŸ”¹ Contoh
Data dikirim dari komputer ke switch lewat **kabel UTP**.

---

# ðŸ“Œ Ilustrasi Alur Data
1. User mengetik `www.google.com` (**Application Layer**).
2. Data dipaketkan dengan TCP/UDP (**Transport Layer**).
3. Paket diberi alamat IP tujuan (**Internet Layer**).
4. Data dikirim lewat kabel/Wi-Fi (**Network Access Layer**).

# ðŸ“˜ 7 Lapisan OSI (Open Systems Interconnection)

Model OSI adalah **kerangka kerja standar** yang menjelaskan bagaimana data dikomunikasikan dari satu perangkat ke perangkat lain dalam sebuah jaringan.  
Terdiri dari **7 lapisan**, dari lapisan paling bawah (fisik) sampai lapisan paling atas (aplikasi).

---

## 1. Physical Layer
ðŸ‘‰ Lapisan paling bawah, berhubungan dengan media transmisi fisik.

### ðŸ”¹ Fungsi
- Mengirimkan bit (0 dan 1) melalui kabel atau sinyal wireless
- Mengatur voltase, kecepatan data, konektor, topologi

### ðŸ”¹ Contoh
- Kabel UTP, Fiber Optic, Wi-Fi
- Hub, Repeater

---

## 2. Data Link Layer
ðŸ‘‰ Menangani **pengiriman frame** antar perangkat dalam satu jaringan lokal (LAN).

### ðŸ”¹ Fungsi
- Mengatur alamat fisik (**MAC Address**)
- Error detection (cek kesalahan saat transmisi)
- Membungkus data menjadi frame

### ðŸ”¹ Contoh
- Ethernet, Wi-Fi
- Switch, Bridge
- Protokol: ARP, PPP

---

## 3. Network Layer
ðŸ‘‰ Menentukan **alamat logis** dan rute perjalanan data.

### ðŸ”¹ Fungsi
- Memberikan **alamat IP**
- Routing data dari pengirim ke penerima
- Mengatur fragmentasi paket

### ðŸ”¹ Contoh
- IP Address (IPv4/IPv6)
- Router
- Protokol: IP, ICMP

---

## 4. Transport Layer
ðŸ‘‰ Mengatur **komunikasi ujung ke ujung** (end-to-end).

### ðŸ”¹ Fungsi
- Memecah data menjadi segmen
- Mengatur port aplikasi
- Menjamin keandalan data (acknowledgment, retransmisi)

### ðŸ”¹ Contoh
- Protokol: **TCP**, **UDP**
- Port: 80 (HTTP), 443 (HTTPS), 25 (SMTP)

---

## 5. Session Layer
ðŸ‘‰ Mengatur **sesi komunikasi** antar aplikasi.

### ðŸ”¹ Fungsi
- Membuka, mengelola, dan menutup sesi
- Sinkronisasi komunikasi antar aplikasi
- Menyediakan checkpoint agar data tidak hilang

### ðŸ”¹ Contoh
- NetBIOS
- RPC (Remote Procedure Call)

---

## 6. Presentation Layer
ðŸ‘‰ Mengatur **format data** agar dapat dimengerti oleh aplikasi.

### ðŸ”¹ Fungsi
- Translasi data (misal: ASCII â†” Unicode)
- Enkripsi & dekripsi data (SSL/TLS)
- Kompresi data

### ðŸ”¹ Contoh
- JPEG, PNG, MP3, MP4
- SSL/TLS

---

## 7. Application Layer
ðŸ‘‰ Lapisan teratas, langsung berinteraksi dengan **user**.

### ðŸ”¹ Fungsi
- Menyediakan layanan aplikasi untuk user
- Menentukan bagaimana aplikasi menggunakan jaringan

### ðŸ”¹ Contoh
- Browser (HTTP/HTTPS)
- Email (SMTP, IMAP, POP3)
- FTP, DNS

---

## 2. DNS (Domain Name System)  

### ðŸ”¹ Pengertian
DNS adalah **sistem yang menerjemahkan nama domain (contoh: www.google.com)** menjadi alamat IP (contoh: 142.250.190.46).  

### ðŸ”¹ Fungsi
- Memudahkan pengguna mengakses situs tanpa menghafal IP Address.  
- Menyediakan layanan pencarian domain secara global.  
- Mempercepat proses koneksi dengan caching DNS.  

### ðŸ”¹ Cara Kerja
1. User mengetik alamat domain di browser.  
2. DNS resolver mencari alamat IP di cache atau meminta ke server DNS.  
3. Server DNS memberikan jawaban berupa IP Address.  
4. Browser menggunakan IP Address untuk menghubungi server tujuan.  

### ðŸ”¹ Contoh
- Domain: `www.youtube.com` â†’ IP: `142.250.72.238`.  

---

## 3. HTTP (Hypertext Transfer Protocol)  

### ðŸ”¹ Pengertian
HTTP adalah **protokol aplikasi** yang digunakan untuk mengirim dan menerima data di web, terutama dokumen HTML, gambar, video, dll.  

### ðŸ”¹ Fungsi
- Menjadi standar komunikasi antara browser (client) dan web server.  
- Mengatur format permintaan (request) dan balasan (response) web.  

### ðŸ”¹ Cara Kerja
1. Client (browser) mengirimkan **HTTP Request** ke server.  
2. Server memproses dan mengirimkan **HTTP Response** (berupa halaman web).  
3. Browser menampilkan hasil ke pengguna.  

### ðŸ”¹ Contoh
- Saat mengetik `http://example.com` di browser, protokol HTTP digunakan.  
- **HTTPS** adalah versi aman (menggunakan SSL/TLS untuk enkripsi).  

---

## 4. FTP (File Transfer Protocol)  

### ðŸ”¹ Pengertian
FTP adalah **protokol standar untuk mengirim dan mengambil file** antara client dan server dalam jaringan.  

### ðŸ”¹ Fungsi
- Mengunggah (upload) file ke server.  
- Mengunduh (download) file dari server.  
- Mengelola file pada server (hapus, ubah nama, pindah folder).  

### ðŸ”¹ Cara Kerja
1. Client menggunakan aplikasi FTP (FileZilla, WinSCP, dll).  
2. Client login ke server FTP dengan username dan password.  
3. File dapat dipindahkan antar client â†” server.  

### ðŸ”¹ Contoh
- Webmaster mengunggah file website ke server hosting menggunakan FTP.  
- Perusahaan mengirim data laporan ke server pusat dengan FTP.
